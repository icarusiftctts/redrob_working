## 1. Project Overview
This is a hackathon submission for the **Redrob Intelligent Candidate Discovery & Ranking Challenge**. The goal is to build a system that ranks ~100,000 candidate profiles against a specific job description (JD) and produces a `submission.csv` containing the top 100 candidates. The project is evaluated through a 5-stage pipeline including format validation, scoring against hidden ground truth, code reproduction, manual review of reasoning, and a final interview.

### Target Role (Job Description)
Senior AI Engineer — Production Embeddings, Retrieval & Ranking.
Key requirements: expertise in embeddings, retrieval, vector search (FAISS, Pinecone, Qdrant, Weaviate), NLP, ranking, recommendation systems, and production deployment.

### Submission Deliverables
1. `submission.csv`: Top 100 ranked candidates.
2. GitHub Repository: Full source code for reproduction.
3. Hosted Sandbox: A working demo (e.g., on HuggingFace Spaces).
4. `submission_metadata.yaml`: Metadata matching the submission template.

## 4. Core Logic & Algorithms

### 4.1 Honeypot Detection (`is_honeypot` in `features.py`)
A static candidate is a synthetic trap inserted by organizers. The current system implements 4 red flags to identify them. If any flag is triggered, the candidate receives a score of -1.0 and is filtered out. These rules are based on programmatical analysis of the data and are finely tuned to catch all 93 ground-truth honeypots without producing false positives.
1.  **Missing Skill Assessment:** A skill present in the `skill_assessment_scores` object is NOT present in the top-level `skills` array.
2.  **Current Job Duration Mismatch:** For the current job, the `duration_months` stated does not match the duration calculated from the `start_date` and the reference date (June 1, 2026), with a tolerance of 2 months.
3.  **Zero-Duration Expert/Advanced Skills:** Any skill with `proficiency` of "expert" or "advanced" but a `duration_months` of 0.
4.  **Stated vs. Calculated Experience Outlier:** The ratio of total career months to stated years of experience is outside the [0.2, 5.0] range.

### 4.2 Feature Extraction & Scoring (`features.py`)
The scoring engine is a weighted linear model with multiplicative modifiers.
The final score for a candidate is: `Final_Score = Base_Score * Behavioral_Multiplier * Location_Multiplier * YoE_Multiplier`.

- **Location Multiplier** (`location_multiplier`): Hard penalty for candidates outside India who won't relocate (0.35x), softer if willing to relocate (0.55x), 1.0 for India-based.
- **YoE Multiplier** (`yoe_multiplier`): Hard penalty for <2yr experience (0.20x), penalties for <5yr (0.50–0.75x), sweet spot 5–12yr (1.0x), heavy penalty for 15yr+ (0.40x).
- **Minimum Evidence Filter** (`has_minimum_evidence`): Requires at least 1 verified core-domain skill (advanced/expert, duration > 6 months). Candidates without this are filtered out entirely.

#### 4.2.1 Feature Groups (22 Features)
- **Skills Group (7 features):** `skill_core`, `retrieval`, `python_sk`, `skill_endorse`, `skill_dur`, `assess`, `github`
- **Career Group (7 features):** `title_curr`, `title_avg`, `consult`, `hop`, `prog`, `product`, `tenure`
- **Semantic Group (3 features):** `sbert`, `bm25`, `headline`
- **Exp/Loc/Edu Group (3 features):** `yoe`, `loc`, `edu`
- **New Signals Group (2 features):** `cert`, `recruit`

#### 4.2.3 Behavioral Multiplier (`compute_behavioral_multiplier`)
A multiplicative factor (clipped to range [0.15, 1.40]) that modifies the base score based on practical data.
- **Profile Completeness:** Penalty for <30% (0.70x), <50% (0.85x), bonus for >=80% (1.05x).
- **Open-to-work (OTW) flag:** Multiplier of 0.55 if not open to work.
- **Profile Recency:** Exponential decay based on `last_active_date` vs. reference date (June 2026). `0.30 + 0.70 * exp(-days / 90)`.
- **Recruiter Response Rate:** Penalty for low response (<0.10 -> 0.45x, <0.25 -> 0.70x, <0.50 -> 0.90x), bonus for high response (>= 0.70 -> 1.10x).
- **Notice Period:** Bonus for 0 days (1.10x), neutral for <=30 (1.00x), penalty for >30.
- **Interview Completion Rate:** `0.50 + 0.50 * completion_rate`.

### 4.3 Semantic Matching (`precompute.py`)
- **Text Source:** Candidate profile text is built from headline, current title, summary, last 5 career history titles/descriptions, and top 20 skills.
- **SBERT:** Uses `all-MiniLM-L6-v2` (22MB model). Computes cosine similarity with the JD text. Scores are min-max normalized to [0, 1].
- **BM25:** Uses `rank_bm25` (BM25Okapi) on tokenized text. Scores are min-max normalized to [0, 1].

### 4.4 Ranking Pipeline (`rank.py`)
1. **Load Candidates:** Reads `candidates.jsonl` and `precomputed/` semantic scores.
2. **Honeypot Filter:** Discards any candidate identified by `is_honeypot()`.
3. **Coarse Scoring:** Uses a lightweight 6-feature model on all non-honeypot candidates to select the top 5000.
4. **Full Scoring:** Runs the full 22-feature model on the top 5000, with behavioral, location, and YoE multipliers.
5. **Sorting & Writing:** Sorts by score descending, then by `candidate_id` ascending for ties. Writes `submission.csv`.

## 5. Feature Functions (Detailed Logic in `features.py`)

### 5.1 Skill Features
- **`f_skill_core_score` (weight 0.18):** Sums weighted core skills (`embedding`, `retrieval`, `vector search`, etc. have weight 3.0; `nlp`, `python` etc. have 2.0; `pytorch` etc. have 1.0; negative weights for wrong domains like `computer vision`). Trust multiplier based on endorsements and duration. Normalized by dividing by 12.0, clamped to [0.0, 1.0].
- **`f_retrieval_explicit` (weight 0.03):** Highest proficiency score for any skill matching `RETRIEVAL_LIBS` (FAISS, Pinecone, etc.).
- **`f_python_explicit` (weight 0.02):** Proficiency score if 'python' is a skill.
- **`f_skill_avg_endorsements` (weight 0.02):** Average endorsements on `advanced`/`expert` skills, normalized to 50.
- **`f_skill_avg_duration` (weight 0.02):** Average duration of core-domain skills, normalized to 60 months.
- **`f_skill_assessment_avg` (weight 0.02):** Average `skill_assessment_scores`, normalized to 100.
- **`f_github_activity` (weight 0.03):** `github_activity_score / 100.0`, or 0.30 if score is -1.

### 5.2 Career Features
- **`f_title_current` (weight 0.07):** 1.0 if current title matches `GOOD_TITLES` (engineer, scientist, ml, ai, nlp, search, etc.), 0.0 if matches `BAD_TITLES` (hr, marketing, sales, etc.), 0.35 otherwise.
- **`f_title_avg` (weight 0.05):** Average title relevance across all career entries.
- **`f_consulting_penalty` (weight 0.03):** Penalizes consulting career history. 0.0 if all consulting. 0.20 if current role is consulting. Otherwise `1.0 - frac * 0.30`.
- **`f_job_hop_penalty` (weight 0.02):** Penalty for short stints (< 18 months). `max(0.30, 1.0 - short_stints * 0.15)`.
- **`f_career_progression` (weight 0.02):** Tracks upward trajectory (junior -> senior -> lead).
- **`f_product_company_frac` (weight 0.02):** Fraction of career months spent at non-consulting companies.
- **`f_relevant_tenure` (weight 0.06):** Total months in ML/AI/NLP roles, normalized to 72 months.

### 5.3 Semantic Features
- **`sbert` (weight 0.08) & `bm25` (weight 0.06):** Precomputed by `precompute.py`. See Section 4.3.
- **`f_headline_keyword_density` (weight 0.05):** Direct keyword match count in headline + summary for JD-specific terms (embedding, retrieval, vector, search, nlp, ranking, etc.). Normalized to 5 hits = 1.0. Complements SBERT/BM25 with exact-match signals.

### 5.4 Exp/Loc/Edu Features
- **`f_years_fit` (weight 0.06):** Bell curve for 5-9 years of experience (sweet spot = 1.0).
- **`f_location_tier` (weight 0.06):** Score based on city tier (Pune/Noida = 1.0, New Delhi = 0.9, etc.).
- **`f_education_score` (weight 0.04):** Based on institution tier, degree level, and CS-field bonus.

### 5.5 New Signal Features
- **`f_certification_score` (weight 0.02):** Scores AI/ML-relevant certifications (AWS ML Specialty, Deep Learning Specialization, NLP, etc.). Generic certs like Scrum/Six Sigma are ignored. Multiple relevant certs give a bonus.
- **`f_recruiter_interest` (weight 0.04):** Social proof signal combining `saved_by_recruiters_30d` (normalized to 12) and `search_appearance_30d` (normalized to 300).

## 7. Submission Requirements & Constraints

- **100 Rows + 1 Header Row**: `candidate_id, rank, score, reasoning`
- **Score Monotonicity**: `score(rank 1) >= score(rank 2) >= ... >= score(rank 100)`
- **Runtime**: <= 5 minutes on CPU with 16GB RAM (for the `rank.py` step).
- **No GPU / Network**: GPU usage and API calls are prohibited during ranking.
- **Honeypot Rate**: Must be < 10% in the top 100.
- **Scoring Metrics**: Composite = `0.50 * NDCG@10 + 0.30 * NDCG@50 + 0.15 * MAP + 0.05 * P@10`.

## 8. Notes for Future Agents/Developers

1. **Weights are Heuristic (v2.0):** The `WEIGHTS` in `features.py` are a starting point. They MUST be tuned using `label.py` + `tune_weights.py`. Do not rely on them for the final submission.
2. **Honeypot Detection is Locked:** The current `is_honeypot` logic catches all 93 ground-truth honeypots. Only modify it if new honeypot patterns are discovered in the dataset (test on a labeled subset first).
3. **Behavioral Multiplier is Aggressive:** The current multiplier can range from 0.15 to 1.40. A candidate with a strong profile but a zero response rate or a very long notice period could be heavily penalized. This is intentional and matches the real-world recruiting logic described in `redrob_signals_doc.docx`.
4. **Reasoning Quality:** This is a human-reviewed stage. The `generate_reasoning` function should provide diverse, specific, honest, and non-template-based justifications. Focus on facts that link back to the JD.
5. **v2.0 Changes Summary:** Fixed critical `yoe_multiplier` bug (was always using 0), negative skill score bug, dead code in `f_consulting_penalty`, added certification/recruiter/headline features, and added profile completeness to the behavioral multiplier. Score range went from [0.09, 0.16] to [0.40, 0.87].

## 9. Quick Start for AI Agents

If you are a new agent taking over this project, here is the exact sequence of commands to run to generate a submission:

1. **Run precomputation** (takes ~20 mins, one-time):
   ```bash
   python precompute.py
   ```
2. **Run the ranker** (takes ~10-20 seconds):
   ```bash
   python rank.py
   ```
3. **(Optional) Validate submit**:
   ```bash
   python validate_submission.py submission.csv
   ```

To improve the model, you must run the labeling and tuning steps:
1. **Label candidates** (interactive):
   ```bash
   python label.py
   ```
2. **Tune weights**:
   ```bash
   python tune_weights.py
   ```
3. **Update `features.py`** with the new weights from the `tune_weights.py` output, then re-run `python rank.py`.
